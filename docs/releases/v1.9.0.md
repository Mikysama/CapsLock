# CapsLock v1.9.0 发布说明

v1.9.0 增加完整的本地生命周期治理，不扩大文件、命令、Web、MCP 或 Skill 权限。

## 初始化、配置与凭据

- `capslock init` 同时支持交互向导和非交互参数；provider 网络测试必须显式启用。
- 配置升级为 version 1，支持 `config validate/migrate`；v1.8 无版本配置备份后保留注释迁移。
- provider 和 MCP 可引用 `env:NAME` 或 `keyring:NAME`；密钥输入不经过 argv，也不会显示在状态和诊断中。

## 备份、导出与诊断

- `backup create/list/verify/restore` 提供本机回滚快照、成员校验和恢复前安全备份。
- `export/import` 提供脱敏 portable archive、确定性冲突映射、跨表引用重写和重复导入幂等性。
- `doctor --json/--strict/--network/--fix` 在配置损坏时仍可运行；写修复保持逐项确认。

## 升级与回滚

workspace 和 memory fresh-v2 schema 从 v2 升至 v3；首次打开前分别使用 SQLite backup API 创建一致性备份。配置 v0 同样先备份再升级。回滚到 v1.8.0 前必须停止全部 CapsLock 进程，同时恢复两套数据库和配置备份；v1.8.0 不能打开 schema v3。

本版本不直接迁移 v1.3.x–v1.7.x 数据库。v1.8 session export v2 和 memory export v3 可由新的顶层 import 接受。

## 安全边界

- portable export 不含模型配置、keyring、`.env`、MCP env 值、embedding 同意或向量。
- 导入的 queued work 不自动启动；旧批准全部失效，历史副作用不能对目标工作区执行 undo。
- lifecycle 操作使用 advisory lock、持久回滚快照和 journal；未完成操作阻止普通启动。

本地发布门禁为 126 项测试通过；确定性评测 10/10、质量 1.0、成本为 0。Linux/macOS、Python 3.11/3.12 CI、密钥扫描和 tag job 仍需在远端通过后才能创建正式 Release。
