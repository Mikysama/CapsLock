# CapsLock v1.5.1 发布说明

v1.5.1 将本地 Skill 简化为 Codex、Claude Code 和 Agent Skills 风格的渐进式 Markdown 工作流。v1.5.0 未正式发布，本版本是 v1.5 系列首个公开 Skill 契约，并彻底移除旧 Skill 运行时和旧包支持。

## Skill 契约

每个 Skill 是一个以 `SKILL.md` 为入口的目录：

```text
skill-name/
├── SKILL.md
├── references/    # 可选
├── assets/        # 可选
└── scripts/       # 可选，只读资源，无执行特权
```

`SKILL.md` 使用 YAML frontmatter 和非空 Markdown 正文。`name` 与 `description` 必填；`license`、`compatibility` 和字符串键值 `metadata` 可选。名称必须与目录一致，并使用 1-64 个小写字母、数字和单连字符分隔的单词。

加载器安全解析 YAML，拒绝重复/未知字段、`allowed-tools`、Claude 行为扩展、符号链接、目录穿越、非 UTF-8 入口和超限包。旧多文件强类型包和旧 Skill 目录不会被发现，也没有自动转换命令；需要保留的工作流必须重新生成为新包。

## 发现与调用

工作区包只从 `.capslock/skills/` 发现，用户包只从 `${CAPSLOCK_HOME:-~/.capslock}/skills/` 发现。工作区包覆盖用户包，禁用状态保存在 `skill_settings`。

普通会话只向模型提供按名称排序、最多 16 KiB 的 Skill 名称和描述。匹配任务后，模型使用只读 `load_skill` 加载正文和资源清单，再用 `read_skill_resource` 分段读取本 run 的 UTF-8 资源快照。二进制资产只出现在清单中，`scripts/` 没有专用执行入口。

显式调用必须位于消息开头：

```text
$workspace-summary focus on release readiness
```

参数保持原始文本并与不可信 Skill 正文一起加入当前普通 run。输出是普通文本并沿用现有 evidence、source 和 memory citations；不创建独立 Skill run，也不要求 JSON 输出。

## 权限与审计

自动和显式 Skill 使用完整普通工具集，唯一授权来源仍是当前 `full_access`、`approve_for_me` 或 `ask_for_approval` 权限模式。Skill frontmatter 不能授予工具、命令、网络、MCP、hooks、subagent 或模型能力。

Skill 自身是持久化指令，因此 `.capslock/skills/**` 的修改在三种权限模式下都需要逐次确认。项目 config、MCP、local 和 state 仍不可由 Agent 修改；state、local、`.env` 和旧运行文件不可读取。

工作区 schema 升至 v5。升级会在自动备份后删除 v1.5.0 开发期的 `skill_runs` 表和历史查询 API，保留 `skill_settings`；新 Skill 调用只写普通 run、tool call、action、event 和 citation 审计。

## 目录升级

项目配置、MCP、Skill、本机私有配置和状态分别位于 `.capslock/config.toml`、`.capslock/mcp.json`、`.capslock/skills/`、`.capslock/local/` 和 `.capslock/state/`。用户级 Skill 和记忆位于 `CAPSLOCK_HOME`。

迁移默认只预览，不加载 `.env`、模型或 SQLite：

```bash
capslock migrate-layout
capslock migrate-layout --scope all --apply --yes
```

旧 config、MCP、state 和 memory 路径继续按布局规则迁移；旧 Skill 路径不再参与发现或迁移。

当前工作区中的旧 `workspace-summary` 包已重新生成为 `.capslock/skills/my-summary/SKILL.md`，删除了 TOML、JSON Schema 和 fixture 文件。其他旧包需要按新契约手动重写。

如果使用在 v1.5.1 之前创建的虚拟环境，需要重新安装项目依赖：

```bash
python -m pip install -e '.[dev]'
```

否则运行时可能报 `ModuleNotFoundError: No module named 'yaml'`。

## 发布检查

新增的 `python -m capslock` 与 `capslock` console script 均应输出相同版本并完成 doctor 启动检查。发布前必须通过完整 pytest、Ruff、依赖检查与审计、仓库卫生、构建、Twine、版本一致性和隔离 wheel 冒烟。正式发布 tag 为 `v1.5.1`，不得补建 `v1.5.0` tag。
