# CapsLock v2.0.0 发布说明

CapsLock v2.0.0 是稳定个人 Agent 里程碑，完成 2.0 前置异步架构后的兼容契约冻结、迁移保障和完整质量门禁。

## 变更

- 删除 1.10.1 标记的 `repositories=`、`bind_run()`、`use_role()`、`summary(run_id)` 和 `max_turns` 过渡入口。
- `WorkspaceAgent.ask_stream()`、`ModelRunSession`、窄端口和 `max_tool_rounds` 成为唯一公开运行时路径。
- 旧配置键和环境变量改为明确错误，不再静默迁移。
- 保持 workspace schema v4、memory schema v3、archive v2、JSONL v2、权限模式和 canonical 布局不变。
- 将最低 Python 版本提升到 3.12，在 Linux/macOS 上执行升级、回滚、恢复和端到端评测门禁。

## 升级

请先阅读 [v2 开发过程与迁移](../development/v2/v2.0.md)，停止运行中的 CapsLock 进程并创建备份。1.10.0 和 1.10.1 工作区在兼容范围内可直接升级；更早版本请先导出或新建工作区后导入数据。

## 安全与已知限制

CapsLock 仍是按需运行的本机 Agent，不包含后台 daemon、远程控制、多 Agent、插件市场或多模态能力。外部模型、Web、MCP 和 embedding 的数据外发仍需要显式配置/同意，所有高风险动作继续经过策略、审批和审计。
