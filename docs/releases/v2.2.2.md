# CapsLock v2.2.2

CapsLock 2.2.2 是内部边界收敛与 inline 交互修复版本。公开的 workspace application、streaming Agent、CLI/TUI、审批和持久化协议保持不变，内部 repository、端口、组合根与运行时按实际能力拆分。

## 架构变化

- 配置拆为类型、document loader、迁移、纯验证、解析规则和 settings resolver；纯验证不访问环境凭据或 keyring。
- Memory repository 拆为 lifecycle、query、source、settings 和 audit；Workflow 拆为 work item、run、journal 与 unit of work，跨表状态转换仍在单一事务内完成。
- Lifecycle 同步门面组合 `BackupService`、`PortableArchiveService`、`ImportCoordinator` 和 `LifecycleIO`；导入映射、引用重写、workflow 归一化与索引重建不再反向依赖 service。
- Action coordinator 使用窄 action/run-state/source 端口，handler 自行 revalidate；MCP stdio 与插件进程由独立 executor 执行。
- `WorkspaceApplication.open()` 使用统一异步资源栈；`ChildAgentRunner`、`ChildCapabilityPolicy`、`RunExecutionService`、`RunOutcomeBuilder`、`RoutePlanner`、`ModelBudgetGate` 和 `RouteAttemptExecutor` 分担运行时职责。
- inline 与 fullscreen 共用 `ForegroundRunController` 和 `AuthorizerBindings`；console script 直接进入 `capslock.cli.app:main`，provider factory 独立于 CLI app。

## Inline 命令树

普通输入、快速连续输入或粘贴、退格和删除现在走同一 completion 刷新路径。输入 `/` 会立即显示完整命令树；继续输入会实时过滤；完整输入 `/status` 等命令时仍保留唯一候选，不再需要先删除字符才能触发显示。

## 兼容与安全

本版本不增加工具、权限、网络范围或持久化字段。workspace schema v5、memory schema v3、config v2、portable archive v2、JSONL v2、CLI 退出码、审批状态机、`WorkspaceApplication.open()/close()` 和 `WorkspaceAgent.ask_stream()` 均保持不变。

## 验证

本地发布门禁包含 Ruff format/check、210 项测试、deterministic Agent 评估、memory 1.00 top-5 hit rate、repository hygiene、wheel/sdist 构建和 Twine 检查。发布矩阵继续面向 Linux/macOS 与 Python 3.12。
