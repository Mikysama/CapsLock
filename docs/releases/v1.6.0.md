# CapsLock v1.6.0 发布说明

v1.6.0 为本地记忆增加受控候选提取和可解释上下文召回。默认 `review` 策略不会直接将模型输出写成可信记忆。

## 用户可见变化

- 成功回答后从用户直接陈述中提取 fact、preference、decision 和 todo 候选。
- 新增 `off`、`review`、`automatic` 策略，以及候选查看、审核、拒绝和清除命令。
- 每轮自动召回最多 5 条相关记忆，并通过 `/memory context` 解释排名。
- 新增可选 FastEmbed 和仅回环 OpenAI-compatible HTTP 嵌入；默认关闭，失败退化到 FTS。
- 记忆导出格式升级到 v2，包含 origin/provenance，并继续导入 v1 文件。

## 安全边界

自动模式不创建 global 记忆，不采纳敏感、冲突、推断或外部来源候选。记忆上下文始终是不可信数据，不能授予权限或覆盖系统规则。项目禁令和本机写入禁用继续优先于自动化策略。

## 升级与降级

首次启动会备份并将用户记忆 schema v1 升级到 v2；工作区 schema 仍为 v5。降级到 v1.5.1 前必须停止所有 CapsLock 进程，并从用户记忆目录的 `backups/` 恢复 schema v1 文件。

FastEmbed 是可选依赖：

```bash
python -m pip install -e '.[local-embeddings]'
```

## 发布检查

正式 tag 为 `v1.6.0`。发布前执行完整测试、Ruff、依赖检查与审计、仓库卫生、构建、Twine、隔离 wheel 冒烟和 `python scripts/verify_release.py --tag v1.6.0`。
