# CapsLock v1.7.1 发布说明

v1.7.1 是 v1.7 工作流 TUI 的兼容性修复版本，重点修复会话恢复、模型输出层级、运行状态反馈和终端启动界面。该版本不扩大工具、权限、网络、MCP 或本地数据范围。

## 会话恢复与管理

- 裸 `capslock resume` 恢复基于 `prompt-toolkit` 的方向键选择器；仍可传入完整 session ID 或唯一前缀。
- 恢复会话时重放已完成消息，以及中断或失败 run 的用户问题和已产生文本。
- 后续模型请求使用恢复后的 session 上下文，并排除当前 run，避免当前问题重复注入。
- `capslock sessions` 支持列表、搜索、重命名、归档、恢复归档、导出和永久删除；删除会同时清理该 session 的关联记录与 session 作用域记忆。

## TUI 输出

- 启动 banner 恢复 `Welcome back`、CapsLock 字符画、运行身份和 Tips；窄终端使用纵向布局，宽终端使用双栏布局。
- reasoning 与最终回答分别使用低对比度暗化斜体和高对比度主文本样式，不再依赖 `Final answer` 字样区分。
- 输入区底部为 Agent 状态预留独立行，思考、文件读取和工具调用期间显示动态字符与对应阶段文本。
- 阶段结束后清除动画，并在 scrollback 中写入静态成功、失败、等待审批或取消结果。
- `--no-spinner`、`--quiet`、`CAPSLOCK_NO_SPINNER=1` 和 `CI=true` 可关闭动态状态。
- 恢复 `/quit` 退出命令，并保留等价的 `/exit`。

## 运行时修复

- OpenAI-compatible adapter 分别解析 reasoning 与回答增量；ToolLoop 不再将二者拼接为同一种输出。
- prompt-toolkit 活动输入与流式输出通过安全终端写入协调，避免刷新覆盖 reasoning、回答或状态结果。
- 每次流仍只产生一个终止事件；JSONL v2、退出码和审批状态语义保持不变。

## 升级边界

v1.7.1 不修改工作区或记忆 SQLite schema，也不改变 session export version 2、memory export version 3 或 JSONL schema version 2。可直接从 v1.7.0 升级，不需要执行数据库迁移。

## 发布检查

版本唯一源已更新为 `1.7.1`。发布前必须通过完整 pytest、Ruff、repository hygiene、依赖检查与审计、构建、Twine、版本一致性和隔离 wheel 冒烟，并使用 `python scripts/verify_release.py --tag v1.7.1` 校验。此次文档更新不创建 Git tag、GitHub Release 或发行包。
