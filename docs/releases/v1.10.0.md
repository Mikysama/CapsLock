# CapsLock v1.10.0 发布说明

v1.10.0 在保留现有审批、取消、检查点和审计边界的同时，为长时间运行增加明确的治理限制。

## 主要更新

- 交互式 TUI 每完成 32 个工具轮次暂停一次，经用户确认后可继续 32 轮。
- `capslock exec` 支持确定性的工具轮次、工具调用数、运行时长、token 和美元费用限制。
- 规范化工具调用连续重复、出现短周期或持续失败时，会在产生新的工具副作用前停止。
- run 治理快照和结构化停止原因可跨 retry 与检查点恢复保留。
- TUI 审批会阻塞当前工具调用，只询问是否执行而不输出动作载荷；批准后，动作在同一个 run 内执行，并将结果返回模型继续推理。
- 裸 `/permissions` 命令会打开交互式权限粒度选择器。

因预算或循环检测停止时，CapsLock 会发出 `stopped` JSONL 终止事件，并返回退出码 `4`。
