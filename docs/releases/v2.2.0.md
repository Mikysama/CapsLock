# CapsLock v2.2.0

CapsLock 2.2.0 新增本机多 Agent 协作。父 Agent 可把独立工作拆成显式子任务，在有界并发下运行，并且只能汇总通过验证的结果。

## 新增

- `delegate_agents` 单层批量委派工具，默认最多四个子任务、并行两个。
- 隔离子工作区、独立 workspace/memory 数据库和默认只读工具目录。
- 任务能力、消息摘要、验证结果审计，以及 `/agents` 查看、取消和清理命令。
- 输出 schema、路径、检查状态、字节上限和 SHA-256 验证。

## 安全边界

子 Agent 不继承父 Agent 的当前权限、记忆、插件工具或运行数据库。写入、命令、Web 和 MCP 必须由任务契约逐项授予并继续通过原有审批链。子 Agent 不能创建孙 Agent，原始子输出始终按不可信数据处理。

## 升级

workspace schema 从 v4 升至 v5，启动时先创建一致性备份再迁移；memory schema v3、portable archive v2、JSONL v2 和 config v2 保持不变。升级前应停止所有 CapsLock 进程并创建备份。

最低 Python 版本仍为 3.12，支持 Linux 和 macOS。

本地发布门禁为 180 项测试通过，确定性 Agent 评测质量 1.0 且无回归，Ruff、依赖审计、构建、Twine、版本一致性和 wheel 导入冒烟通过。正式 `v2.2.0` tag 和 GitHub Release 仍需等待 Linux/macOS 远端 CI。
