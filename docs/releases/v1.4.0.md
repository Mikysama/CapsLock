# CapsLock v1.4.0 发布说明

v1.4.0 增加用户可控的本地记忆数据层和手动工作流。记忆不会从会话自动生成，也不会默认注入模型上下文。

## 主要更新

- 用户级 SQLite 记忆库支持 global、workspace、session 作用域、固定类型、来源、置信度和过期时间。
- `/memory` 提供列表、全文搜索、详情、新增、编辑、遗忘、撤销、永久清除、导入导出和写策略管理。
- 模型可按需使用只读 `search_memories` / `get_memory`，最终回答通过 memory citation 展示来源。
- FTS5 提供本地检索，并为短 CJK 等分词缺口提供安全 substring 回退；不使用外部向量服务。
- 敏感内容保存前自动脱敏；事件、工具审计和导出不包含原始命中值。

## 升级与回滚

首次打开 v1.3.2 工作区数据库时，CapsLock 在 `.capslock/backups/` 创建 schema v1 备份，再升级到 schema v2 并为新消息关联 run ID。失败会回滚并停止启动；重复启动不会重复备份。

用户记忆保存在独立的用户数据目录。回滚到 v1.3.2 前应退出 CapsLock，并从备份恢复工作区 schema v1 数据库；v1.3.2 不会读取用户记忆库，记忆文件可保留供再次升级。

## 已知边界

- 不自动提取、写入或召回记忆。
- 不提供向量检索、云同步、跨设备共享或外部嵌入。
- 永久清除不可撤销；导出文件由用户负责保管和删除。

## 发布检查

本地 Python 3.12/Linux 已通过 74 项测试，隔离依赖审计未发现已知漏洞。发布使用 `v1.4.0` tag；Linux/macOS、Python 3.11/3.12 必须通过测试、ruff、依赖审计、构建、twine、wheel 安装冒烟、仓库卫生和版本一致性检查后，CI 才创建 GitHub Release。
