# CapsLock v2.2.1

CapsLock 2.2.1 是 CLI/TUI 前端优化版本。默认界面继续运行在普通终端主缓冲区，保留原生 scrollback，同时采用组件化消息、工具活动、队列、状态和输入设计；第一版 Textual 全屏界面继续通过显式参数提供。

## 界面变化

- 裸 `capslock` 与 `capslock resume` 默认使用 `inline`；可通过 `--ui fullscreen` 或 `CAPSLOCK_UI=fullscreen` 启用保留的全屏界面，命令行参数优先。
- inline Composer 不固定在窗口底部，而是出现在最新上下文之后并随输出移动。欢迎 banner 保持完整尺寸，输入框保留完整上下边框且不显示冗余快捷键提示。
- 用户消息、reasoning、最终回答、工具活动和系统结果使用分层样式；最终回答支持 Markdown、代码、表格和终端链接，连续只读/搜索工具默认合并折叠。
- fullscreen 保留固定输入区和可滚动 transcript；`/` 命令候选改为纵向完整列表，可滚动并自动跟随 `↑/↓` 选中项，不再横向溢出或截断为前 8 项。
- 两种界面的容器、消息、输入区、代码样式和模态框均不设置固定背景色，沿用用户终端背景。

## 安全与兼容

工具运行事件增加可选 `presentation.version = 1` 展示摘要，只从 allowlist 字段生成并执行脱敏和长度限制。交互审批在本机显示动作类型、风险、目标，以及最多 40 行、4 KiB 的脱敏命令或 diff；默认选择仍为拒绝，原始参数、完整输出、文件正文和凭据不写入展示事件。

本版本不增加工具、授权级别或后台能力。workspace schema v5、memory schema v3、portable archive v2、JSONL v2、配置 v2、权限状态机、`capslock exec` 文本/JSONL 输出和退出码均保持兼容。Textual 初始化失败会明确提示使用 `--ui inline`，不会静默切换界面。

## 验证

本地发布验证包括 Ruff、201 项测试、版本一致性检查，以及 `capslock-2.2.1` wheel 和 sdist 构建。TUI 测试覆盖 120×32、80×24、60×20 布局、长历史分页、流式回答、默认拒绝审批、透明背景和完整纵向命令列表。
