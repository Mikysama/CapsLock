# CapsLock v2.3.1

CapsLock 2.3.1 是 CLI/TUI 显示与 fullscreen 性能补丁。它统一用户上下文消息的视觉层级，并消除 fullscreen 空闲状态下持续刷新终端造成的高 CPU 和交互卡顿。

## 显示优化

- inline 与 fullscreen 的用户 prompt 行使用 `#E0E0E0` 浅灰背景。
- 背景紧贴左侧蓝色标记并铺满消息宽度，多行 prompt 保持一致布局。
- Markdown、Composer、系统消息和其余容器继续继承终端默认背景。

## Fullscreen 性能

- ActivityBar 定时器在空闲时暂停，只在 spinner 可见时运行。
- spinner 刷新率从 30 FPS 降至 12 FPS；清空后的 ActivityBar 不再重复提交相同内容。
- transcript 同步跳过未变化的 `MessageWidget`，流式 token 只更新当前发生变化的消息。
- 在 199×60 终端实测中，完成 run 后的空闲 CPU 从约 30% 降至约 0.8%，终端写入从约 30 次/秒降至正常光标闪烁所需的约 2 次/秒。

## 兼容性

本补丁不改变 Tool Runtime、权限、配置、数据库、memory、portable archive、JSONL 或插件协议。当前格式仍为 config 5、workspace schema 8、memory schema 3、portable archive 3、JSONL schema 3 和 plugin manifest/protocol/grant 4。

## 发布验证

2.3.1 发布候选通过 Ruff、全量 pytest、版本一致性、repository hygiene 与构建产物检查。fullscreen 回归测试覆盖空闲定时器暂停、ActivityBar 去重、消息增量更新、用户 prompt 背景宽度、Composer 与 Markdown 透明渲染。
