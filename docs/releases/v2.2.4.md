# CapsLock v2.2.4

CapsLock 2.2.4 升级运行内核与本机安全边界，同时保持产品定位不变：本机前台运行、固定命令模板和单层子 Agent。

## 运行与上下文

- `AgentSession.run_stream(RunRequest)` 成为唯一执行入口，同一 session 串行运行并完整传播取消。
- 工具增加 JSON Schema、capability、并发与结果大小契约；并发安全只读调用有界并行，checkpoint 保持调用顺序。
- 上下文改为 token 感知预算，支持结构化 compaction、digest 复用、确定性兜底和失败熔断。
- 大型工具结果外置为 session 隔离 artifact，提供校验、大小上限和分块读取。

## 插件与审批

- 插件默认使用 Linux Bubblewrap 或 macOS sandbox profile；缺少 sandbox backend 时拒绝执行。
- 双向 capability broker 对 workspace、network、固定命令和命名 credential 重新执行范围检查、审批、脱敏和审计。
- 文件写入审批展示真实 diff；credential 值不持久化，插件回显也会在进入日志或 action result 前脱敏。
- `trusted-native` 需要逐工作区显式确认，每次调用仍强制人工审批。

## 事件与存储

- 事件增加 event ID 和 trace ID；UI 立即消费，耐久 sink 批量写入并在终止前强制 flush。
- SQLite 使用单 writer 和只读连接池，复合状态转换继续由事务独占。
- Workspace schema 6 增加 context compaction、tool artifact 和 tool invocation 元数据。
- CLI 只通过 `WorkspaceQueries` 读取状态，不再访问 session repository bundle。

## 协议与兼容边界

当前格式为 config 3、workspace schema 6、memory schema 3、JSONL 3、portable archive 3 和 plugin protocol 3。旧配置、数据库、archive、JSONL、插件协议和已删除 Python 接口不提供运行时兼容或转换入口。升级已有数据必须在部署前离线备份、转换并校验。

## 验证

发布门禁包含 compileall、Ruff、全量 pytest、repository hygiene、确定性评测、依赖审计、构建、Twine、版本一致性和隔离 wheel 冒烟。详细实现与部署要求见 [当前开发者文档](../development/v2/current.md)。
