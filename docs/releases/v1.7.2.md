# CapsLock v1.7.2 发布说明

v1.7.2 是会话管理体验补丁，重点改进历史会话的交互式删除流程。该版本不扩大工具、权限、网络、MCP 或本地数据范围。

## 交互式删除

- 新增 `capslock session` 作为 `capslock sessions` 的兼容别名，现有复数命令保持有效。
- 运行 `capslock session delete` 且不提供 Session ID 时，列出最近的会话历史，并支持方向键导航和 Enter 选择。
- 选定后显示将永久删除的会话标题与短 Session ID，再请求 `y/n` 二次确认。
- 输入 `y` 执行删除；输入 `n` 或直接按 Enter 返回会话选择器，可重新选择。
- 显式传入完整 Session ID 或唯一前缀时保留原有单次确认流程；`--yes` 仍可跳过二次确认。

## 兼容与数据边界

永久删除仍会清理该会话的消息、run、工具调用、引用、动作和 session 作用域记忆。运行中或等待审批的会话仍必须先取消相关任务。

v1.7.2 不修改工作区或记忆 SQLite schema，也不改变 session export version 2、memory export version 3 或 JSONL schema version 2。可直接从 v1.7.1 升级，不需要执行数据库迁移。

## 发布检查

版本唯一源已更新为 `1.7.2`。发布提交前必须通过完整 pytest、Ruff、repository hygiene、依赖检查与审计、构建、Twine、版本一致性和隔离 wheel 冒烟，并使用 `python scripts/verify_release.py --tag v1.7.2` 校验。
