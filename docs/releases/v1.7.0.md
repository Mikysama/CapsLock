# CapsLock v1.7.0 发布说明

v1.7.0 引入流式工作流 TUI、可恢复运行和完整会话治理，同时保持 v1.6 的工具、权限和本地数据边界。

## 用户可见变化

- TTY 中裸 `capslock` 默认进入行内 TUI；`capslock chat --classic` 可使用旧界面。
- 计划、前台队列、工具状态、待审批动作和 token 用量持续可见。
- `capslock exec [PROMPT] [--json]` 提供稳定脚本入口；`ask` 保持兼容。
- 会话支持全文搜索、归档、恢复、JSON+Markdown 导出和永久删除。
- 失败运行可从最近工具轮次检查点恢复，已完成副作用不会重放。
- 工具调用达到 `max_turns` 后仍保留一次最终答案合成机会，避免长任务在完成读取后无法返回总结。
- 默认工具调用轮次从 6 提高到 32；仍可通过 `CAPSLOCK_MAX_TURNS` 或项目配置覆盖。
- 恢复会话时重放完整消息与失败/取消 run 的可见部分输出；流式正文使用 prompt-toolkit 安全刷新，不再被活动输入区覆盖。
- `Thinking...` 使用动态字符状态指示；CapsLock 仍不展示模型隐藏思维链。

## 升级与降级

首次启动会备份 workspace 数据库并从 schema v5 升至 v6；用户记忆 schema 保持 v2。降级到 v1.6.0 前必须停止所有 CapsLock 进程，并从 `.capslock/state/backups/` 恢复 schema v5 数据库。v1.6.0 不应直接打开 schema v6。

## 安全边界

非交互模式不会请求或推断审批。导出默认脱敏，且不包含系统提示、原始检查点或其他作用域记忆正文。永久删除不可撤销，并同时清理当前 session 的记忆、候选和召回数据。

## 发布检查

正式 tag 为 `v1.7.0`。发布前执行完整测试、Ruff、依赖检查与审计、仓库卫生、构建、Twine、隔离 wheel 冒烟和 `python scripts/verify_release.py --tag v1.7.0`。
