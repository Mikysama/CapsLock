# CapsLock v1.8.0 发布说明

v1.8.0 增加可复现的多模型路由、真实外部 embedding、模型费用预算和端到端评测，不扩大文件、命令、Web、MCP 或 Skill 权限。

## 模型路由与费用

- 支持多个 OpenAI-compatible provider 和模型 profile，分别配置 `reasoning`、`fast`、`embedding` 和预留的 `vision` 角色。
- 路由记录候选、排除原因、选择、重试和降级；timeout、429 和 5xx 最多重试两次，流式输出开始后禁止重放。
- 只有 data-policy 相同的显式后备模型可以自动接管。
- 每次调用及 run/session 聚合记录 token、延迟、错误和美元费用。TUI 可批准下一次超预算调用；`exec` 确定性返回 `model_budget_exceeded`。

## 外部嵌入

`/memory embeddings enable external <model-profile>` 在任何网络请求前展示外发字段、记录数和 UTF-8 字节数，并说明后续召回会发送查询文本。确认绑定 provider、模型和 data-policy；配置变化或撤销后确认失效。外部服务失败继续使用 FTS。

## 升级与回滚

workspace 和 memory fresh-v2 schema 均从 1 升至 2。首次启动会在各数据库旁的 `backups/` 下生成一致性备份，然后事务迁移；失败时回滚并报告备份路径。旧 application ID 和未知 schema version 不会自动转换。

回滚到 v1.7.2 前必须停止所有 CapsLock 进程，并分别恢复升级前的 workspace 和 memory 备份；v1.7.2 无法打开 schema v2 数据库。

## 评测与已知边界

- 完整测试为 120 项，确定性核心评测覆盖八类场景并通过 10 个参数化用例，质量 1.0、测试 API 成本为 0。
- 本期只正式支持 OpenAI-compatible 接口；原生 provider 和视觉输入不在范围内。
- token 预检使用 UTF-8 载荷的保守估算，最终计费以 provider usage 为准。
- 真实 provider 评测仅在显式提供凭据时运行，结果为发布候选的非阻断采样。

版本唯一源已更新为 `1.8.0`。创建 `v1.8.0` tag 前仍需通过全部 CI、依赖审计、构建、Twine、隔离 wheel 冒烟和版本一致性检查。
