# CapsLock v1.4：本地记忆基础

## 目标与边界

v1.4 提供由用户显式管理、可检索、可撤销和可清除的本地记忆。模型只能按需读取可见记忆；本版本不从对话自动提取记忆，不自动写库，也不在每轮 prompt 中自动召回。

## 数据与作用域

- 用户级独立 SQLite 保存 `global`、`workspace` 和 `session` 记忆，并使用工作区指纹与会话 ID 做隔离。
- 记录正文、固定类型、来源、置信度、过期时间、revision 和状态；FTS5 只索引活动且未遗忘的正文。
- `forget` 从检索和模型工具隐藏但保留可撤销历史；`purge` 在安全删除模式下清除正文、来源引用、FTS 和历史版本，仅保留无正文审计元数据。
- 工作区 schema v2 为消息增加 run ID。模型读取记忆时记录 revision；记忆改变、过期或失效后，相关历史 run 不再进入未来模型上下文。

## 安全与运维

- add、edit 和 import 在持久化前脱敏 secret 字段、Bearer token、私钥和已知 token 格式；事件和工具审计不保存正文或搜索查询。
- `[memory] enabled=false` 是项目级写入禁令，本机开关不能覆盖；读取和显式导出仍可用。
- 导入导出限制在工作区 JSON，拒绝符号链接越界；导入有大小/数量限制并全量校验后原子提交。
- 工作区 schema v1 升级到 v2 前自动备份；用户记忆库保留独立 schema 版本和迁移前备份能力。

## 后续版本

候选提取、自动策略、相关性召回、冲突合并和嵌入检索仍属于 v1.6.0，不进入 v1.4.0。
