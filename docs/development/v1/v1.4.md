# CapsLock v1.4：本地记忆与会话历史

## 目标与边界

v1.4 提供由用户显式管理、可检索、可撤销和可清除的本地记忆。模型只能按需读取可见记忆；本版本不从对话自动提取记忆，不自动写库，也不在每轮 prompt 中自动召回。

## 数据与作用域

- 用户级独立 SQLite 保存 `global`、`workspace` 和 `session` 记忆，并使用工作区指纹与会话 ID 做隔离。
- 记录正文、固定类型、来源、置信度、过期时间、revision 和状态；FTS5 只索引活动且未遗忘的正文。
- `forget` 从检索和模型工具隐藏但保留可撤销历史；`purge` 在安全删除模式下清除正文、来源引用、FTS 和历史版本，仅保留无正文审计元数据。
- 工作区 schema v2 为消息增加 run ID。模型读取记忆时记录 revision；记忆改变、过期或失效后，相关历史 run 不再进入未来模型上下文。

## 安全与运维

- add、edit 和 import 在持久化前脱敏 secret 字段、Bearer token、私钥和已知 token 格式；事件和工具审计不保存正文或搜索查询。
- `[memory] enabled=false` 是项目级写入禁令，本机开关不能覆盖；读取和显式导出仍可用。
- 导入导出限制在工作区 JSON，拒绝符号链接越界；导入有大小/数量限制并全量校验后原子提交。
- 工作区 schema v1 升级到 v2 前自动备份；用户记忆库保留独立 schema 版本和迁移前备份能力。

## 后续版本

候选提取、自动策略、相关性召回、冲突合并和嵌入检索仍属于 v1.6.0，不进入 v1.4.x。

## v1.4.1：会话历史管理补丁

v1.4.1 在 v1.4.0 的存储和 CLI 边界上补充会话历史管理，不改变记忆读取策略、模型工具集合、权限模式或外部访问范围。

### 标题模型

- `sessions` 增加 `title`、`title_source` 和 `title_updated_at`；`title_source` 取值为 `pending`、`first_question` 或 `manual`。
- 新会话先使用带创建时间的临时标题。创建首个 run 时，将第一个非空问题合并空白并截断到 80 字符，原子写为 `first_question` 标题。
- `/rename <title>` 和 `capslock sessions rename <session-id-or-prefix> <title>` 写入 `manual` 标题；后续 run 只更新 `pending` 标题，因此不会覆盖用户命名。
- 标题是可重复的展示属性，不进入模型上下文，也不参与会话归属判断；完整 Session ID 始终是唯一标识。

### schema v3 迁移

- 首次打开 schema v2 工作区数据库前，沿用 SQLite backup API 在 `.capslock/backups/` 创建一次备份。
- 迁移增加三个标题字段，并按 `started_at` 读取最早 run 的问题回填旧会话；没有有效问题的记录保留临时标题。
- 迁移在现有事务边界内完成，失败会回滚并保留备份；schema v3 重复启动不会再次创建迁移备份。
- v1.4.0 拒绝读取更新的 schema v3。回滚到 v1.4.0 前必须退出进程并恢复 schema v2 备份。

### CLI 交互

- `capslock` 默认进入新聊天，`capslock chat` 继续可用；`--debug` 和 `--workspace` 的全局参数语义不变。
- `capslock sessions` 按 `updated_at` 倒序显示标题、更新时间和 12 位 ID 前缀，默认限制 20 条。
- 裸 `capslock resume` 使用 `prompt-toolkit` 原生 choice 控件，支持上下方向键和 Enter；`--limit` 控制候选数量。
- 恢复列表根据终端显示宽度计算列宽，补偿选中符号和序号前缀，并用显示宽度而非字符数对齐 CJK 标题。
- `capslock resume <session-id-or-prefix>` 保持非交互兼容；前缀必须在当前工作区数据库中唯一。

### 空会话清理

- 聊天循环使用统一 `finally` 路径处理 `/exit`、`/quit`、EOF、输入阶段 Ctrl-C 和异常退出。
- 删除使用单条条件 SQL，要求标题仍为 `pending`，并且不存在 run、消息、动作、任务或来源。
- 清理前额外查询 session 级记忆；只要存在活动、遗忘或过期的 session 记忆，就保留会话记录。
- 手动重命名本身会将标题来源改为 `manual`，即使尚未提问也不会被视为完全空会话。

### 验证与边界

- 迁移测试覆盖 schema v2 标题回填、无问题回退、备份和 schema 版本升级。
- CLI 测试覆盖裸启动、显式 `chat`、两种重命名、完整/短 ID 恢复、空列表、方向键选择和中英文列对齐。
- 退出测试覆盖 `/exit`、`/quit`、EOF、Ctrl-C，以及内容、手动标题和 session 记忆的保留条件。
- v1.4.1 不实现跨工作区历史、标题搜索、归档、导出或批量删除；这些能力仍属于后续工作流 TUI 里程碑。
