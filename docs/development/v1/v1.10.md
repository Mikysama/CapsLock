# CapsLock v1.10：运行预算与循环治理

v1.10 将工具循环从单一轮次错误升级为可恢复、可审计的多维运行治理。

## 运行契约

- `max_tool_rounds` 表示返回工具调用的模型往返轮次，默认 32；`max_turns` 仅保留兼容别名。
- TUI 达到软限制后每次可续期 32 轮，停止时使用无工具模型请求生成总结。
- `capslock exec` 可限制轮次、工具调用数、时长、token 和美元费用；任何限制耗尽返回退出码 4。
- retry 和检查点恢复沿父 run lineage 继承累计用量、预算和循环历史。

## 循环治理

工具名称和脱敏、排序后的参数生成稳定指纹。连续相同调用、2--4 长度的重复周期和持续失败重试在执行前被停止；参数递进的分页、范围和批量读取不视为重复。

## 数据迁移

workspace schema 从 v3 升至 v4，增加 run 治理快照、工具尝试指纹、预算事件和 `stopped` 状态。迁移前自动备份并回填旧 run 计数；portable archive v2 仍接受 v1.9 archive v1。memory schema 保持 v3。

## 非目标

不增加后台 daemon、退出后续跑、无限运行、自动批准或非工具语义循环检测。
