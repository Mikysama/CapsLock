# CapsLock v1.7：工作流 TUI

## 目标与边界

v1.7 将同步聊天界面升级为流式、可取消、可恢复的终端工作流。它不增加工具或权限范围，不启动 daemon，前台进程退出后不会继续执行队列。

## 运行契约

- 每个用户请求创建持久化 work item；run 是一次执行尝试，重试通过 parent run 和稳定步骤关联。
- 模型文本、工具状态、等待审批和终态以顺序事件输出；非流式 provider 自动退化为单个文本增量。
- `max_turns` 只计算包含工具调用的模型轮次；达到上限后额外允许一次最终答案合成，但该轮不得再调用工具。
- 完整模型回答或成功工具调用后保存消息检查点。恢复从最近完整检查点继续，不重放已成功副作用。
- 取消会结束 run、work item 以及关联的非终态动作；命令继续使用进程组终止和强制清理兜底。

## TUI 与 CLI

- TTY 中裸 `capslock` 默认启动 prompt-toolkit 行内 TUI，保留 scrollback；`--classic` 保留旧界面。
- 状态区展示最近计划、前台队列和 token；运行时可继续输入排队，`/queue move|cancel` 调整未开始工作，`/retry` 从稳定点创建新尝试；`/approvals` 统一展示风险、原因与回滚建议。
- `capslock exec` 提供纯文本和版本化 JSONL；非交互待审批返回退出码 3。
- 会话支持当前工作区内全文搜索、归档、恢复、JSON+Markdown 导出和永久删除。

## 数据与迁移

workspace schema v6 增加 `work_items`、`run_steps`、`run_events`、session FTS，以及 run/action/task 扩展列。v5 升级前自动备份，迁移失败回滚。用户记忆 schema 保持 v2；永久删除会通过两阶段流程清理 session 级记忆后再删除 workspace 记录。

## 验证

测试覆盖事件顺序、流式降级、取消、风险快照、退出码、稳定点不重放、搜索/归档/导出/删除和 80/120/160 列布局，并保留全部旧版本安全回归。
