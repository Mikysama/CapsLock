# CapsLock v1.8：模型路由与评测

## 目标与边界

v1.8.0 在不改变工具和权限语义的前提下，将单模型客户端升级为可审计的 OpenAI-compatible 多 provider 路由，并加入外部 embedding 同意、模型费用预算和固定评测。本期不接入原生 provider 或视觉输入，也不实现 v1.10 的工具轮次、时长和循环预算。

## 路由与故障契约

调用方显式选择 `reasoning`、`fast`、`embedding` 或 `vision` 角色。路由按配置顺序过滤上下文不足、凭据缺失和有限预算下无单价的 profile，并保存全部候选、排除原因和最终选择。timeout、429 与 5xx 最多重试两次；仅向 data-policy 相同的显式候选降级。首个流式 delta 后的故障结束当前 run，不重放可见文本或工具调用。

每个调用保存 provider、model、role、attempt、延迟、usage、费用和错误。run/session 预算在下一次请求前按实际消耗、估算输入和最大输出预留检查；TUI 可批准单次超限调用，非交互 `exec` 硬停止。

## 外部嵌入与迁移

外部 embedding 只能引用 `[routing].embedding` 中的 profile。启用前展示 `memory.content`、未来 `recall.query`、当前记录数和 UTF-8 字节数；确认绑定 provider、remote model 和 data-policy。配置变化、撤销或缺失确认时零网络请求，服务故障继续退化到 FTS。

workspace 和 memory fresh-v2 schema 从 1 升到 2。相同 application ID 自动创建 SQLite 一致性备份并事务迁移；旧 application ID 与未知版本继续拒绝。回滚到 v1.7.2 前必须停止所有进程，并恢复迁移前的两套备份。

## 评测与验证

`python scripts/evaluate_agent.py --mode deterministic` 覆盖工作区问答、编辑、执行、Web、MCP、记忆、Skill 和模型路由，并对质量、p95 延迟和成本基线判定回退。`--mode live` 仅在显式提供凭据时执行真实 provider 非阻断采样。
