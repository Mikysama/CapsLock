# CapsLock v2.1：本地工具插件 SDK

## 范围

v2.1 只支持用户从本地目录显式安装的工具插件。插件不能注册 UI、模型 Provider、生命周期 Hook、后台任务、Skill 或自定义 Action Handler，也不能从 URL、Git、PyPI 或市场安装。

插件属于用户主动安装的受信本地代码。独立进程隔离插件崩溃、超时、输出和依赖边界，但不承诺阻止恶意插件直接访问本机资源。

## 包与 Manifest

插件目录必须包含 `capslock-plugin.toml`：

```toml
manifest_version = 1
protocol_version = 1
name = "example-plugin"
version = "1.0.0"
description = "Example local tool plugin"
requires_capslock = ">=2.1.0,<3.0.0"
entrypoint = ["plugin.py"]
permissions = ["workspace_read"]

[[tools]]
name = "echo"
description = "Return the supplied text"
parameters = { type = "object", properties = { text = { type = "string" } }, required = ["text"], additionalProperties = false }
```

支持的权限为 `workspace_read`、`workspace_write`、`network`、`process` 和 `credentials`。权限用于风险披露、工作区授权和审计，不是 OS 沙箱声明。

安装器拒绝符号链接、特殊文件、缺失入口、未知字段、重复工具、路径逃逸和超限包，并为全部相对路径与内容计算 SHA-256。

## SDK 与协议

Python 插件可以使用公开的 `capslock.plugin_sdk`：

```python
from capslock.plugin_sdk import ToolDefinition, serve_plugin


def echo(arguments):
    return {"text": arguments["text"]}


serve_plugin(
    tools=[
        ToolDefinition(
            "echo",
            "Return the supplied text",
            {
                "type": "object",
                "properties": {"text": {"type": "string"}},
                "required": ["text"],
                "additionalProperties": False,
            },
            echo,
        )
    ]
)
```

stdio 协议使用一行一个 JSON 对象，`protocol_version = 1`，并支持 `initialize`、`list_tools` 和 `call_tool`。stdout 只能包含协议响应，诊断输出必须写入 stderr。宿主会校验响应 ID、协议版本、工具集合、超时和输出大小。

模型可见名称为 `plugin_<plugin-name-with-underscores>_<tool-name>`。插件调用使用现有 `mcp_call` 外部动作持久化通道，因此 workspace schema v4、JSONL v2 和权限状态机保持兼容。

## 安装与授权

- 用户安装目录：`${CAPSLOCK_HOME:-~/.capslock}/plugins/<name>/<version>/`。
- 用户注册表：`${CAPSLOCK_HOME:-~/.capslock}/state/plugins.json`。
- 用户审计日志：`${CAPSLOCK_HOME:-~/.capslock}/state/plugin-audit.jsonl`。
- 工作区授权：`.capslock/local/plugins.json`，不进入 portable archive。

升级先验证源包和协议，再复制到暂存目录并原子发布。升级后的摘要或版本与原工作区授权不一致时，插件保持禁用，必须重新确认权限。

## 运行与安全

插件进程使用包目录作为 cwd，只继承 `PATH`、UTF-8 和协议相关环境变量，不继承模型或 CapsLock 凭据。每次调用受 MCP 配置中的超时和输出上限约束；取消、超时和协议错误会终止进程。

每次调用进入 `ActionCoordinator`，按高风险第三方本地工具处理。批准后若安装摘要或工作区授权已经变化，执行会失败并要求重新提案。

## 验证

```bash
python -m ruff format --check .
python -m ruff check .
python -m pytest -q
```

插件回归测试覆盖 manifest 安全、协议错误、安装与升级原子性、摘要篡改、工作区重新授权、审批、超时、取消、输出上限和审计。
